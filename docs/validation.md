# Validation

`qcow2-core` and `qcow2-forensic` parse untrusted QCOW2 images — a format that
QEMU/KVM/libvirt write and that can carry backing files, internal snapshots,
encryption headers, and QEMU's own dirty/corrupt flags. Correctness is therefore
established the way forensic tooling must be: against an **independent oracle** (a
different tool that already decodes the same bytes correctly) on **real
images** with known construction — not against fixtures we hand-encoded and then
graded ourselves.

This page records exactly which oracle and which corpus back each capability, so
the claim is independently re-checkable. Per-file corpus provenance (source,
download URL, license, hash) lives in
[`core/tests/data/README.md`](https://github.com/SecurityRonin/qcow2-forensic/blob/main/core/tests/data/README.md);
the fleet-wide machine index is `issen/docs/corpus-catalog.md` (entry B1). This
page cross-references both rather than duplicating them.

## How to read the evidence tiers

Each validation below is tagged with the trustworthiness of its check, not
whether the data is "synthetic":

- **Tier 1** — an independent third party authored the artifact *and* the answer
  key, or it is real-world data decoded by an independent tool. The strongest claim.
- **Tier 2** — real engine output whose ground truth is derivable from the
  documented construction, or confirmed by an *independent code path* on real
  data. Genuinely checked, but we chose the scenario.
- **Tier 3** — fixture and expected answer both authored here, nothing
  independent vouching. Used only for per-branch coverage, never as a
  correctness claim: a self-consistent round trip proves internal consistency,
  not correctness against real-world bytes.

## Independent oracles

| Oracle | Independent of us? | Validates | Tier |
|---|---|---|---|
| **`qemu-img convert`** (QEMU 11.0.0) | Yes — separate C codebase (the reference QCOW2 implementation) | `Qcow2Reader` virtual-disk bytes match `qemu-img`'s decode, byte-for-byte at sampled offsets | 1 (CirrOS) / 2 (synthetic source) |
| **`qemu-img create` / `snapshot` / `convert`** (QEMU 11.0.0) | Yes — QEMU writes the images we then inspect | `inspect()` / `snapshots()` / `refcount_report()` / `audit_path()` against real qemu-authored backing-file, snapshot, encryption, v1, and clean images | 1 |
| **`flate2`** | Yes — vetted third-party DEFLATE codec we reuse | The zlib/DEFLATE cluster decode itself (a maintained, audited codec) | 1 |

`qemu-img` is both an oracle (its `convert -O raw` output is the answer key for
the reader) and a corpus generator (its `create`/`snapshot` build the
feature-carrying images the inspector and auditor are checked against). The
`qemu-img`-gated tests skip silently when `qemu-img` is absent rather than
failing, so a minimal CI runner without QEMU still passes the committed tests.

## Per-capability validation

### Virtual-disk read (uncompressed + zlib clusters, sparse zeros) — Tier 1

`core/src/lib.rs::corpus_cirros_reads_match_qemu_raw_convert` (`core/src/lib.rs:500`,
gated on `qemu-img` + the committed CirrOS image) decodes the **real CirrOS
0.6.3** image and compares the bytes at samples spanning the MBR, partition
table, multiple cluster boundaries, mid-image, and the last sector against
`qemu-img convert -O raw`. Matching QEMU's own decode of an image QEMU's build
system wrote establishes correctness against real-world bytes across zlib-cluster
decoding, L1→L2 indirection, and unallocated (zero) regions. The zlib decode
itself is the vetted third-party `flate2` crate (`core/src/lib.rs:182`).

A companion differential, `reads_match_qemu_raw_convert` (`core/src/lib.rs:452`),
round-trips a deterministic 1 MiB pattern through `qemu-img convert -O qcow2` and
checks the reader against the known source bytes at sector/cluster boundaries.
Because the scenario is one we chose, this is **Tier 2** (real engine output,
ground truth derived from the construction).

### `inspect()` header facts — Tier 1

`core/tests/real_images.rs` validates the lenient header inspector against
**real qemu-img-produced images**, each carrying one forensic feature:

| Image (qemu-img) | Asserted `inspect()` fact | Test (`core/tests/real_images.rs`) |
|---|---|---|
| `create -b base.qcow2 -F qcow2` overlay | `has_backing_file == true`; backing **name** contains `base.qcow2`, backing **format** == `qcow2`; base has neither | `:28`, `:139` |
| `snapshot -c snap1` | `snapshot_count >= 1` | `:64` |
| `-o encrypt.format=luks` | `encryption_method != 0` | `:223` |
| `create -f qcow` (v1) | `version == 1` | `:121` |
| CirrOS 0.6.3 (clean) | v2/v3, no backing, `encryption_method == 0`, size > 0 | `:251` |

The encryption (LUKS) and v1 tests skip cleanly when the local QEMU build lacks
those writers, so the absence of a capability never silently passes.

### Snapshot enumeration — Tier 1

`snapshots_enumerates_named_snapshot_on_real_image` (`core/tests/real_images.rs:185`)
creates two named snapshots (`alpha`, `beta`) with `qemu-img snapshot -c` and
asserts both names are enumerated, each with a non-empty id and a plausible
recent Unix timestamp.

### Refcount / orphaned-cluster report — Tier 1

`refcount_report_on_real_clean_image_finds_no_orphans` (`core/tests/real_images.rs:88`)
converts 4 MiB of real content into QCOW2 with `qemu-img convert` and asserts the
report reads QEMU's default 16-bit refcounts (`refcount_order == 4`), locates a
non-zero refcount table, counts allocated data clusters, and finds **zero
orphaned clusters** on a freshly-converted clean image. The expected values are
derived from QEMU's documented defaults, not hardcoded to a captured fixture.

### Auditor findings on real images — Tier 1

`forensic/tests/real_images.rs` drives the full `audit_path()` pipeline on real
qemu-authored images: `audit_path_names_the_backing_file_on_real_overlay`
(`:26`) confirms the backing-file finding names `base.qcow2`, and
`audit_path_surfaces_per_snapshot_findings_on_real_image` (`:67`) confirms one
per-snapshot finding for each of `alpha` and `beta`.

### Auditor grading + header parsing — Tier 3 (self-authored, per-branch coverage)

The anomaly-to-`Finding` mapping, severity grades, and anomaly codes
(`QCOW2-BACKING-FILE`, `QCOW2-ENCRYPTED`, `QCOW2-INTERNAL-SNAPSHOTS`,
`QCOW2-DIRTY`, `QCOW2-CORRUPT`, `QCOW2-EXTERNAL-DATA`) are exercised against
crafted `Qcow2Info` values in `forensic/tests/audit.rs`, and the lenient header
parser is exercised against hand-built v3 headers in `core/tests/inspect.rs`
(clean, all-flags-set, non-QCOW2). These are **Tier 3**: we authored both the
fixture and the expected answer, so they prove the code's internal consistency
and per-branch coverage, not correctness against real-world bytes. The real-image
tests above are the independent correctness claim that backs the same code paths.

The `ZERO_PLAIN` zero-cluster path (`zero_plain_cluster_reads_as_zeros`,
`core/src/lib.rs:427`) is also a self-authored regression fixture (Tier 3): it
patches `L2[0] = 1` (QCOW_OFLAG_ZERO) and asserts the read returns zeros rather
than seeking to file byte 1.

### Robustness — never panic, never over-read

The reader and auditor are read-only and panic-free by construction:
`#![forbid(unsafe_code)]`, no `unwrap`/`expect`/unchecked indexing in production
parsers, every length and offset bounds-checked. Four `cargo-fuzz` targets drive
crafted bytes through the entry points with the invariant "must not panic":
`fuzz_open` (`Qcow2Reader::open`), `fuzz_read` (open + read the whole virtual
stream), `fuzz_inspect` (lenient header inspection), and `fuzz_forensic` (the
full `inspect → audit` pipeline). A `fuzz.yml` CI workflow builds and smoke-runs
each target.

## Reproducing the validation

The committed tests run with `cargo test`. The `qemu-img`-gated tests need QEMU on
`PATH` at `/opt/homebrew/bin/qemu-img`; they skip silently otherwise. The CirrOS
corpus is committed, so the corpus differential runs wherever `qemu-img` exists.

```sh
# Everything (committed CirrOS corpus + qemu-img-gated real-image tests)
cargo test --workspace

# Reader differential vs qemu-img convert (synthetic source + real CirrOS)
cargo test -p qcow2-core reads_match_qemu_raw_convert
cargo test -p qcow2-core corpus_cirros_reads_match_qemu_raw_convert

# inspect() / snapshots() / refcount_report() vs real qemu-img images
cargo test -p qcow2-core --test real_images

# audit_path() vs real qemu-img images
cargo test -p qcow2-forensic --test real_images

# Re-fetch the CirrOS corpus if absent
curl -L -o core/tests/data/cirros-0.6.3-x86_64-disk.img \
  https://download.cirros-cloud.net/0.6.3/cirros-0.6.3-x86_64-disk.img

# Optional CORPUS_DIR-based corpus job (sparse + cirros)
CORPUS_DIR=/path/to/corpus cargo test -p qcow2-core --test corpus
```

## Coverage & fuzzing as backstops

Line coverage is enforced in CI (`cargo llvm-cov`, failing on any zero-hit line
not annotated `// cov:unreachable`), and the four fuzz targets enforce the
no-panic invariant on crafted bytes. Both are regression backstops that prove
behavior is exercised — they are not the correctness claim. The `qemu-img` and
`flate2` oracles above are.
